# 🔄 Как обновить бота на GitHub и Render

## Способ 1: Через веб-интерфейс GitHub (проще всего)

### Обновить один файл:

1. Откройте ваш репозиторий: https://github.com/fashffffffff/ChatTweaksBotTG
2. Нажмите на файл, который нужно изменить (например, `runtime.txt`)
3. Нажмите значок **карандаша** (Edit this file) справа вверху
4. Внесите изменения
5. Внизу страницы нажмите **"Commit changes"**
6. В появившемся окне нажмите **"Commit changes"** еще раз
7. **Готово!** Render автоматически обнаружит изменения и передеплоит бота (2-3 минуты)

### Загрузить новые файлы:

1. Откройте ваш репозиторий
2. Нажмите **"Add file"** → **"Upload files"**
3. Перетащите файлы из `c:/projects/bot`
4. Нажмите **"Commit changes"**
5. Render автоматически обновится

### Удалить файл:

1. Откройте файл в репозитории
2. Нажмите значок **корзины** (Delete this file) справа вверху
3. Нажмите **"Commit changes"**

## ⚡ Что происходит после обновления?

1. Вы делаете изменения на GitHub
2. Render **автоматически** видит изменения
3. Render **автоматически** запускает новый деплой
4. Через 2-3 минуты бот обновлен и работает

**Вам НЕ нужно:**
- Заходить на Render
- Нажимать какие-то кнопки
- Перезапускать вручную

---

## 📊 Как проверить статус деплоя?

1. Откройте [Render Dashboard](https://dashboard.render.com)
2. Найдите ваш сервис `telegram-mention-bot`
3. Во вкладке **"Events"** увидите:
   - "Deploy started" - деплой начался
   - "Deploy live" - бот обновлен и работает
4. Во вкладке **"Logs"** увидите:
   - Процесс установки
   - "Бот запущен..." - всё работает

---

## 🐛 Если что-то пошло не так:

### Бот не обновляется:
- Подождите 5 минут
- Проверьте вкладку "Events" на Render
- Если деплой failed - смотрите логи

### Ошибка в логах:
- Проверьте синтаксис в `bot.py`
- Убедитесь, что `requirements.txt` правильный
- Проверьте `runtime.txt` - должно быть `python-3.11.9`

### Бот не отвечает:
- Подождите 30 секунд (может просыпаться)
- Проверьте логи на Render
- Убедитесь, что деплой успешен (зеленая галочка)
